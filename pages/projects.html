My Computer Science Projects

Student Enrollment Application

Description: A full-stack web application that reduced course enrollment processing from 30 days to under an hour for ~1,000 students across 24 districts served by Ulster BOCES. The system accepts Excel uploads containing student enrollment preferences, stores data in MS SQL Server, runs a demand-based seat-allocation algorithm, and automatically generates enrollment reports per course and district — replacing a manual Excel-based workflow. Deployed to Windows Server 2022 in a production environment.

Technologies: Java, Spring Boot, Spring Security, MS SQL Server, Thymeleaf, HTML, CSS, JavaScript, JdbcTemplate, Apache POI, IIS, WinSW

Key Features & Implementation:

  • Authored a demand-based seat-allocation algorithm that aggregates course preference counts across all districts, distributes seats proportionally by demand, fills remaining seats via round-robin, and rebalances students evenly across course sections.
  • Implemented a JdbcTemplate data access layer with a multi-district schema (one SQL Server table per district), using parameterized queries and batch operations for CRUD operations across ~1,000 student records.
  • Engineered a lazy DataSource that defers SQL Server connection setup to login time, dynamically configuring the connection from credentials supplied through the login form.
  • Leveraged Apache POI (XSSFWorkbook) to parse and validate uploaded Excel files and generate post-enrollment reports per course and district.
  • Secured the application with Spring Security: CSRF token validation, BCrypt-hashed credentials, one concurrent session per user, session invalidation on logout, and HSTS headers.
  • Deployed Spring Boot as a Windows Service via WinSW with IIS configured as an HTTPS reverse proxy using ARR and URL Rewrite for SSL termination.

Roomsy (Roommate Finder)

Project URL: Roomsy (Roommate Finder)

Description: A cross-platform mobile application (Android/iOS) that connects users with compatible roommates based on lifestyle preferences, interests, and location. The app uses a comprehensive profile system where users specify their preferences, habits, and desired living arrangements. Now released on both the App Store and Google Play Store.

Technologies: Flutter, Dart, Firebase, Google Cloud Vision API, MVVM Architecture

Key Features & Implementation:

  • Integrated Firebase Cloud Functions with Google Cloud Vision API for automated content moderation of user-uploaded profile images before storage.
  • Architected Firebase authentication flow with email verification, Apple Sign-In, and Google Sign-In for seamless and secure user access.
  • Implemented Cloud Firestore for secure user data management with role-based access control and data encryption.
  • Implemented cloud functions for account lifecycle management, including cascading deletion of user data and assets upon account removal.
  • Developed multi-tier caching strategy (in-memory, Firestore, and Firebase Storage) to minimize API costs and improve response times.
  • Following MVVM architecture principles for clean separation of concerns and maintainable codebase.
